### 오늘 배운 것
* 절차지향 프로그래밍 VS 객체지향 프로그래밍
  1. 절차지향 (Procedural Programming)
     : 프로그램을 순서대로 명령하는 **절차(과정)** 중심

     - 대표 언어 : C, Pascal 등
     - 장점 : 구조가 단순, 직관적
     - 단점 : 프로그램이 커질수록 복잡해짐
     - 작은 규모의 프로그램에 적합
     - 데이터와 함수가 분리되어 있음
    
  2. 객체지향 (Object-Oriented Programming)
     : **객체 (Object)** 라는 단위로 데이터를 묶고, 그 객체가 자신의 **기능 (메서드)** 을 갖도록 설계

     - 대표 언어 : Java, Python, C++, C# 등
     - 장점 : 코드 재사용 가능, 유지보수 용이
     - 단점 : 처음 구조 설계가 복잡할 수 있음
     - 대규모 프로젝트에 적합
     - 계산을 객체가 스스로 처리함

💁‍♀️ 예를 들어, 절차지향은 요리사가 직접 레시피를 찾아 하나하나 따라하는 방식 / 객체지향은 요리를 잘하는 셰프에게 "요리해줘"하고 맡기는 방식
