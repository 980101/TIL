* Java 파일이 실행되는 과정
1. .java 파일을 만든다.
2. Javac 컴파일러가 .java 파일을 바이트코드로 변환한다. -> .class 파일 생성
3. JVM (Java Virtual Machine)이 바이트코드를 읽어서 실행

( *.java 파일은 src, .class 파일 = 바이트코드는 bin에 저장됨.)

___

* 객체 생성 시, new 키워드를 왜 써야할까?
: 사용 가능한 실체(=객체)를 만들기 위해 메모리에 올려야 하기 때문
```java
Student s = new Student();
```
1. new Student()
</br>→ Student 클래스 기반으로 새 객체를 생성하고
</br>→ JVM의 힙(heap) 메모리에 올림

2. Student s = ...
</br>→ 생성된 객체의 주소(참조값)를 변수 s에 저장.
</br>→스택에 저장됨

___

*  Annotation(애너테이션)
: 자바 클래스, 메서드, 변수 등에 의미 있는 정보를 부여하는 특수한 문법
```java
@Override
public String toString() {
    return "Hello";
}
```
위 코드에서 @Override는 “이 메서드는 상위 클래스의 메서드를 재정의하고 있어요”라고 컴파일러에게 알려주는 것

___

* Maven
: 프로젝트 관리 및 빌드 자동화 도구 → 자바 프로젝트의 컴파일, 실행, 패키징, 라이브러리 관리를 자동으로 해준다.

| 기능               | 예시                                           |
| ---------------- | -------------------------------------------- |
| 📦 라이브러리 자동 다운로드 | `pom.xml`에 입력만 하면 필요한 jar 파일 자동으로 다운받음       |
| 🔨 프로젝트 빌드       | `mvn compile`, `mvn package`로 컴파일, 실행 파일 생성  |
| 🧪 테스트 실행        | `mvn test`로 단위 테스트 자동 실행                     |
| 🏗 표준 프로젝트 구조 제공 | `src/main/java`, `src/test/java` 같은 폴더 자동 설정 |
| ♻️ 반복 작업 자동화     | 설치, 정리, 배포 등을 명령어 한 줄로 처리 가능                 |
