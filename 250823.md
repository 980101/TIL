### 오늘 배운 것

#### 오버헤드
: 어떤 작업을 수행하기 위해 추가로 드는 시간, 메모리, 자원

<br>

- 프로그래밍에서의 오버헤드

| 종류           | 예시                                             |
| ------------ | ---------------------------------------------- |
| **메모리 오버헤드** | 데이터를 저장하기 위해 필요한 추가 구조 (ex. 포인터, JVM 내부 데이터 등) |
| **시간 오버헤드**  | 추가로 처리해야 하는 로직, 함수 호출, 인터프리터 실행 등              |
| **시스템 오버헤드** | 운영체제가 관리하기 위해 사용하는 CPU, 메모리, I/O 리소스 등         |

<br>

- Java에서의 오버헤드

백준에서처럼 Java를 사용할 때 : Java는 C나 C++처럼 바로 실행되는 게 아니라
<br> **JVM(Java Virtual Machine)** 이라는 프로그램 위에서 돌아간다.
<br>이 JVM은 클래스 로딩, 메모리 관리(GC), 스레드 관리 등을 자동으로 해줌
<br><br>📌 이 JVM이 실행되면 내부적으로 이미 수십 MB의 메모리를 점유 → JVM 오버헤드

<br>
