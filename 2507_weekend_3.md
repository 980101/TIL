### 오늘 배운 것
* stage : Git의 개념 중 하나인 Staging Area(임시 저장 영역) ➡️ 커밋할 준비가 된 파일 목록에 올리는 것
* workspace (Eclipse) : 사용자가 작업하는 모든 프로젝트, 설정, 환경을 저장하는 폴더
  * 역할

  
  | 기능              | 설명                                               |
  | --------------- | ------------------------------------------------ |
  | 📁 **프로젝트 관리**  | 워크스페이스 안에 여러 개의 프로젝트를 만들고 관리함                    |
  | ⚙️ **설정 저장**    | 이클립스의 사용자 설정 (에디터 환경, 폰트, 자동 저장, 플러그인 설정 등)을 저장함 |
  | 🧠 **메타데이터 저장** | `.metadata` 폴더에 이클립스 내부 상태, 빌드 정보, 로그 등을 저장함     |
  | 💾 **로컬 히스토리**  | 파일 변경 이력도 잠깐 저장해서 버전 되돌리기 가능                     |

   * 주의사항
      - .metadata 폴더는 절대 삭제하면 안된다. → 삭제 시 프로젝트는 사라지진 않지만 설정, 환경 정보가 초기화된다.
      - 서로 다른 버전의 Eclipse에서 같은 워크스페이스를 쓸 경우 충돌이 생길 수 있으므로 주의해야 한다.
