### 오늘 배운 것
* 문자열 다루기
  * 문자형 데이터는 ''로 감싸진 형태 필수. "a"의 경우는 문자열로 취급한다.
  * 줄바꿈은 \n, 큰따옴표는 \" 형태로 표현하면 정상적으로 출력된다. 이러한 경우는 문자의 역할에서 벗어난다는 의미에서 escape 문자라고 함.
* 변수의 정의
  * 변수 : 바뀔수 있는 값 의미. <변수에 올 수 있는 데이터타입> <변수이름>으로 표현.
  * 변수에 데이터 타입을 지정하는 것은 데이터를 쓸 때 마다 타입이 무엇인지 확인할 수고를 덜기 위함.
  * 다른 사람이 코드를 볼 때 의미를 빠르게 파악할 수 있도록 하는 수단 중 하나. 코드를 읽기 쉽게 작성하는 것은 매우 중요.
 
* 데이터 타입의 변환
  * 데이터타입 변수명 = (변환하려는 데이터 타입) 데이터값 형태로 표현 -> 값의 손실이 있는 경우, 명시적으로 컨버트 / 값의 손실이 없는 경우, 자동으로 컨버트

  ```java
  
  double a = 1;
  System.out.println(a);  // 1.0 출력. 자동으로 컨버트된 경우

  int b = 1.1;
  System.out.println(b);  // 오류. 값의 손실 0.1이 있기 때문

  int c = (int)1.1;
  System.out.println(c);  // 1 출력. 명시적으로 컨버트한 경우
 
  ```

  * 숫자형 데이터를 문자열 데이터로 변환
  ```java
  String str = Integer.toString(10);  // "10"
  ```

* JPMS 도입 이후, 모든 패키지가 모듈에 의해 명시적으 선언되지 않으면 접근할 수 없게 됨. (JDK 9이후)
  * 모듈(module) : 자바 패키지들의 패키지. 코드의 재사용성을 높이기 위해 추상화 단계를 하나 더 추가한 것
