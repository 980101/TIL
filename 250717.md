### 오늘 배운 것
* 접근 제어자
  : Java에서 접근 범위를 제어하는 키워드

| 접근 제어자   | 접근 범위                    | 설명                                           |
|----------------|-------------------------------|------------------------------------------------|
| public      | 어디서든 접근 가능            | 다른 클래스, 다른 패키지에서도 접근 가능       |
| private      | 같은 클래스      | 외부 클래스나 자식 클래스에서도 접근 불가      |
| protected   | 같은 패키지, 자식 클래스      | 상속 관계에 있는 다른 클래스에서도 접근 가능    |
| (default)    | 같은 패키지         | 접근 제어자 생략 시 적용, 외부 패키지에선 불가  |

    - public : 외부에서 자유롭게 쓰게 하고 싶을 때 사용
    - private : 내부적으로만 쓰고 외부에는 숨기고 싶을 때 (보안, 안정성 위해)

💁‍♀️ 멤버변수는 private로 지정하는 것이 좋다. </br>
객체지향 프로그래밍(OOP)의 핵심 원칙 중 하나인 캡슐화 가능. </br>
객체의 내부 상태를 외부에서 직접 접근하지 못하게 하고, getter / setter 같은 메서드를 통해 안전하게 접근하도록 유도
